# 重複菜名處理說明

## 問題背景

在 `dish.csv` 中，同一個菜名可能對應多個 `dish_id`。這通常是因為：

1. **不同價位**：例如「招牌紅蟳米糕」有小份、中份、大份
2. **套餐組合**：例如「招牌紅蟳米糕(78)」、「招牌紅蟳米糕(128)」等不同套餐
3. **不同規格**：例如「炭烤野生烏金2兩」、「炭烤野生烏金4兩」

## 範例

從 `dish.csv` 中可以看到：

```csv
dish_id,name_zh
142,招牌紅蟳米糕
348,招牌紅蟳米糕
371,紅蟳米糕（乙支）
521,招牌紅蟳米糕
577,特製紅蟳米糕
603,特製紅蟳蒸米糕
634,經典紅蟳米糕
932,招牌紅蟳米糕
933,特製紅蟳米糕
```

這裡「招牌紅蟳米糕」對應了 4 個不同的 dish_id：142, 348, 521, 932。

## 處理策略

### 舊版本（錯誤）

舊版本只會為第一個 dish_id 寫入描述，其他的會被跳過：

```
招牌紅蟳米糕 -> 只更新 dish_id=142
dish_id=348, 521, 932 都沒有描述 ❌
```

### 新版本（正確）

新版本會為所有對應的 dish_id 寫入**相同的描述和標籤**：

```
招牌紅蟳米糕 -> 更新 dish_id=142, 348, 521, 932
所有這些 dish_id 都會有相同的描述和標籤 ✅
```

## 實作細節

### 1. CSV 映射改為 List

```python
# 舊版本
mapping: Dict[str, int] = {}
mapping["招牌紅蟳米糕"] = 142  # 只保留一個

# 新版本
mapping: Dict[str, List[int]] = {}
mapping["招牌紅蟳米糕"] = [142, 348, 521, 932]  # 保留所有
```

### 2. 處理時遍歷所有 dish_id

```python
# 舊版本
dish_id = name_to_id.get(name_zh)  # 只取一個 int
dish = find_dish_by_id(session, dish_id)
upsert_detail_for_dish(session, dish, desc, tags_str)

# 新版本
dish_ids = name_to_id.get(name_zh)  # 取得 List[int]
for dish_id in dish_ids:  # 遍歷每個 dish_id
    dish = find_dish_by_id(session, dish_id)
    upsert_detail_for_dish(session, dish, desc, tags_str)
```

## 為什麼需要這樣處理？

### 使用場景 1：用戶搜尋

當用戶搜尋「紅蟳米糕」時，可能會找到多個 dish_id：
- dish_id=142：單點價格 $XXX
- dish_id=348：套餐A包含
- dish_id=521：套餐B包含

所有這些都應該顯示相同的菜餚描述和標籤。

### 使用場景 2：推薦系統

根據標籤推薦菜品時，應該涵蓋所有相關的 dish_id，而不是只推薦某一個。

### 使用場景 3：統計分析

分析「海鮮類」菜品時，所有紅蟳米糕的變體都應該被計入。

## 注意事項

1. **描述一致性**：所有重複菜名的 dish_id 都會有完全相同的描述
2. **標籤一致性**：所有重複菜名的 dish_id 都會有完全相同的標籤
3. **更新效率**：如果一個菜名對應 5 個 dish_id，就會寫入 5 筆記錄

## 特殊情況

如果某些 dish_id 實際上應該有**不同的描述**（例如規格差異），建議：

1. 在 JSON 中使用更精確的菜名（包含規格）
2. 或在 CSV 中調整菜名以區分不同規格
3. 或在後續手動調整特定 dish_id 的描述

例如：
```json
{
  "dish_name": "炭烤野生烏金（2兩）",
  "description": "...",
  "tags": [...]
}
```
